Project: Train a Quadcopter How to Fly¶

Design an agent to fly a quadcopter, and then train it using a reinforcement learning algorithm of your choice!

Try to apply the techniques you have learnt, but also feel free to come up with innovative ideas and test them.

Controlling the Quadcopter¶

We provide a sample agent in the code cell below to show you how to use the sim to control the quadcopter. This agent is even simpler than the sample agent that you'll examine (in agents/policy_search.py) later in this notebook!

The agent controls the quadcopter by setting the revolutions per second on each of its four rotors. The provided agent in the Basic_Agent class below always selects a random action for each of the four rotors. These four speeds are returned by the act method as a list of four floating-point numbers.

For this project, the agent that you will implement in agents/agent.py will have a far more intelligent method for selecting actions!

import random

class Basic_Agent():
    def __init__(self, task):
        self.task = task
    
    def act(self):
        new_thrust = random.gauss(450., 25.)
        return [new_thrust + random.gauss(0., 1.) for x in range(4)]

labels = ['time', 'x', 'y', 'z', 'phi', 'theta', 'psi', 'x_velocity',

          'psi_velocity', 'rotor_speed1', 'rotor_speed2', 'rotor_speed3', 'rotor_speed4']
results = {x : [] for x in labels}

# Run the simulation, and save the results.
with open(file_output, 'w') as csvfile:
    writer = csv.writer(csvfile)
    writer.writerow(labels)
    while True:
        rotor_speeds = agent.act()
        _, _, done = task.step(rotor_speeds)
        to_write = [task.sim.time] + list(task.sim.pose) + list(task.sim.v) + list(task.sim.angular_v) + list(rotor_speeds)
        for ii in range(len(labels)):
            results[labels[ii]].append(to_write[ii])
        writer.writerow(to_write)
        if done:
            break

The Task¶

A sample task has been provided for you in task.py. Open this file in a new window now.

The __init__() method is used to initialize several variables that are needed to specify the task.

  • The simulator is initialized as an instance of the PhysicsSim class (from physics_sim.py).
  • Inspired by the methodology in the original DDPG paper, we make use of action repeats. For each timestep of the agent, we step the simulation action_repeats timesteps. If you are not familiar with action repeats, please read the Results section in the DDPG paper.
  • We set the number of elements in the state vector. For the sample task, we only work with the 6-dimensional pose information. To set the size of the state (state_size), we must take action repeats into account.
  • The environment will always have a 4-dimensional action space, with one entry for each rotor (action_size=4). You can set the minimum (action_low) and maximum (action_high) values of each entry here.
  • The sample task in this provided file is for the agent to reach a target position. We specify that target position as a variable.

The reset() method resets the simulator. The agent should call this method every time the episode ends. You can see an example of this in the code cell below.

The step() method is perhaps the most important. It accepts the agent's choice of action rotor_speeds, which is used to prepare the next state to pass on to the agent. Then, the reward is computed from get_reward(). The episode is considered done if the time limit has been exceeded, or the quadcopter has travelled outside of the bounds of the simulation.

In the next section, you will learn how to test the performance of an agent on this task.

The Agent¶

The sample agent given in agents/policy_search.py uses a very simplistic linear policy to directly compute the action vector as a dot product of the state vector and a matrix of weights. Then, it randomly perturbs the parameters by adding some Gaussian noise, to produce a different policy. Based on the average reward obtained in each episode (score), it keeps track of the best set of parameters found so far, how the score is changing, and accordingly tweaks a scaling factor to widen or tighten the noise.

Define the Task, Design the Agent, and Train Your Agent!¶

After specifying your task, use the sample agent in agents/policy_search.py as a template to define your own agent in agents/agent.py. You can borrow whatever you need from the sample agent, including ideas on how you might modularize your code (using helper methods like act(), learn(), reset_episode(), etc.).

Note that it is highly unlikely that the first agent and task that you specify will learn well. You will likely have to tweak various hyperparameters and the reward function for your task until you arrive at reasonably good behavior.

As you develop your agent, it's important to keep an eye on how it's performing. import sys
import numpy as np
import pandas as pd
from task import Task
from agents.agent import DDPG

import matplotlib.pyplot as plt
%matplotlib inline

num_episodes = 3000
target_pos = np.array([10., 10., 50.])
task = Task(target_pos=target_pos, runtime=10.0)
agent = DDPG(task)

episodes = []

for i_episode in range(1, num_episodes+1):
    total_reward = 0
    best_reward = -np.inf
    worst_reward = np.inf
    best_distance_to_target = np.inf
    mean_distance_to_target = 0
    steps = []
    distances = []
    step_count = 0
    
    # start a new episode
    state = agent.reset_episode()
    
    while True:
        # Determine action and reward based on state
        action = agent.act(state)
        next_state, reward, done = task.step(action)

        # Track rewards and results for analysis
        steps.append((state, action, reward))
        total_reward += reward
        best_reward = max(best_reward, reward)
        worst_reward = min(worst_reward, reward)
        distance_to_target = task._distance_to_target()
        distances.append(distance_to_target)
        best_distance_to_target = min(best_distance_to_target, distance_to_target)
        mean_distance_to_target = (step_count * mean_distance_to_target + distance_to_target) / (step_count + 1)
        step_count += 1
        
        # Take action
        agent.step(action, reward, next_state, done)
        state = next_state
        
        # Handle end of episode
        if done:
            episodes.append((total_reward, 
                             best_reward, 
                             worst_reward,
                             best_distance_to_target,
                             mean_distance_to_target,
                             distances,
                             len(steps), 
                             steps,
                             best_distance_to_target <= 15)) # Success condition
            print("\rEpisode = {:4d}, total_reward = {:7.3f} (best = {:7.3f}, worst = {:7.3f}, best_distance_to_target = {:7.3f}, , mean_distance_to_target = {:7.3f})".format(
                i_episode, total_reward, best_reward, worst_reward, best_distance_to_target, mean_distance_to_target), end=" - ")  # [debug]
            break
            
    # If last 10 episodes were success, stop exploring
    last_10_successes = [success for _, _, _, _, _, _, _, _, success in episodes[-10:]]
    print(f"{sum(last_10_successes)} of last 10 episodes succeeded", end="")
    if sum(last_10_successes) == 10:
        break
        
    sys.stdout.flush()

Reflections¶

Question 1: Describe the task that you specified in task.py. How did you design the reward function?

Answer: I chose to adapt the task to represent flying to a point in 3D space. I experimented with a lot of different approaches for the reward function. I ended up using a reward function based on the inverse of the distance to the target, $d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2 + (z_2 - z_1)^2}$. In order to increase the gradient of positive reward as the copter approached the target, I ended up squaring the inverse. Additionally, it then seemed like the copter would frequently crash, so I started adding a negative reward for flying lower than 10 units, but ultimately knocked that down to be lower than 5.

I experimented with a few other approaches, like punishing if the angle of the copter was more than 30 degrees. This didn't seem to help much and the copter was still able to learn good flight patterns without explicitly being rewarded on that component.

Question 2: Discuss your agent briefly, using the following questions as a guide:

  • What learning algorithm(s) did you try? What worked best for you?
  • What was your final choice of hyperparameters (such as $\alpha$, $\gamma$, $\epsilon$, etc.)?
  • What neural network architecture did you use (if any)? Specify layers, sizes, activation functions, etc.

Answer: I initially started exploring a deep Q-learing agent, but once I realized I was working with a continuous action space based on rotor speeds, I switched to using a Deep Deterministic Policy Gradient agent. This agent uses an actor-critic approach to learning both the Q function and optimal policy.

Working with the provided outline of the DDPG agent, I modified the buffer size to $10^6$ and decreased the $\tau$ to $10^{-3}$ and left $\gamma$ at 0.99 and batch size at 64.

For the actor network, I kept the same 32, 64, 32 sizes for the hidden layers, though I did explore using the 400, 300 architecture outlined in the paper "Continuous Control With Deep Reinforcement Learning" (https://arxiv.org/pdf/1509.02971.pdf), but went back to the simpler network to decrease training time. I was able to still get good results. I also added batch normalization to each of these layers.

For the critic, Q, network, I kept the 32, 64 layers for both the state and action pathways. I also added batch normalization and L2 regularization to this network.

I maintined relu activation on all networks.

Question 3: Using the episode rewards plot, discuss how the agent learned over time.

  • Was it an easy task to learn or hard?
  • Was there a gradual learning curve, or an aha moment?
  • How good was the final performance of the agent? (e.g. mean rewards over the last 10 episodes)

Answer: I don't think the task was that hard to learn once I zeroed in on a decent reward function. It always initially started with consistantly low scores as the copter would often just crash. As it learned to actually fly and learned that it could continue getting positive rewards while aloft (until the episode runtime expired), it pretty quickly found the target spot.

As you can see from the graph above, once it learned to fly, it quickly began maximizing the reward until it met the success criteria of flying to within 15 units of the target point for 10 consecutive episodes, at which point, it was terminated.

When looking at the results and actual position of the successful episodes, I see that future improvements could be made to the agent by only terminating the training after reaching a success criteria of remaining within a certain, and maybe smaller, radius from the target point. The successful episodes above, while flying through the target point (or close to), it seems the copter would still explore the space beyond and above the target point. I'm confident that further trials and allowing it longer training would hone this.

Question 4: Briefly summarize your experience working on this project. You can use the following prompts for ideas.

  • What was the hardest part of the project? (e.g. getting started, plotting, specifying the task, etc.)
  • Did you find anything interesting in how the quadcopter or your agent behaved?

Answer: Overall, I think the hardest part was dealing with the slow feedback cycle that's common in any sort of deep learning project. I waited and waited to see if the copter would ever figure out how to fly. Growing frustrated, I kept tweaking hyperparamters in my reward function to punish crashing or flying low. For awhile, though, this actually led to it crashing faster since it was picking up negative rewards and for flying low and the best way to maximize the reward was to crash quickly. Adjusting the parameters, I was able to reduce this behavior.

The coolest part of this project for me was getting a stronger intuitive sense of how to design a reward function that the networks could learn the right behaviour and making sure the graidents for different points in the state space had the right levels.

And oh, when it actually starts learning and consistently flys to the target is such a cool moment to watch. Great project.
